# 🐐 Записной Козёл

Премиальная карточная игра-PWA «Записной Козёл». Тёмный luxury-дизайн, золотые
акценты, стеклянные панели, плавные анимации, игра против ботов трёх уровней,
статистика и гибкие настройки. Готова к публикации в App Store / Google Play
как PWA.

## ✨ Что внутри

- **Стек:** React 18 + TypeScript + Vite + Tailwind CSS + Framer Motion + Zustand.
- **PWA:** офлайн-кэш, манифест, установка на домашний экран iPhone/Android.
- **Экраны:** Главный, Игра, Правила, Статистика, Настройки + экраны
  победы/поражения и итогов раунда.
- **Игровой движок:** вся логика правил в `src/game/` (UI только отображает).
- **Боты:** 3 уровня сложности (Лёгкий / Средний / Сложный).
- **Сохранение:** статистика и настройки в `localStorage`.

## 🚀 Как запустить (для новичка)

1. Установи [Node.js](https://nodejs.org) (LTS) — если ещё не стоит.
2. Открой терминал в папке проекта и выполни по очереди:

```bash
npm install      # установить зависимости (один раз)
npm run dev      # запустить игру в режиме разработки
```

3. В терминале появится адрес, например `http://localhost:5173/`.
   Открой его в браузере — игра запустится.
4. Нажми **«Играть»** → играй против ботов.

### 📱 Как открыть на телефоне

- Телефон и компьютер должны быть в **одной Wi-Fi сети**.
- После `npm run dev` Vite покажет строку **Network: http://192.168.x.x:5173/** —
  открой этот адрес в браузере телефона.
- Чтобы установить как приложение: в Safari (iPhone) → «Поделиться» →
  «На экран „Домой"»; в Chrome (Android) → меню → «Установить приложение».

### 🏗️ Сборка для публикации

```bash
npm run build    # собрать продакшн-версию в папку dist/
npm run preview  # локально проверить собранную версию
```

Папку `dist/` можно залить на любой хостинг (Vercel, Netlify, GitHub Pages) —
это и будет устанавливаемое PWA.

## 📦 Android APK (скачать и установить)

Приложение упаковано в нативный Android-проект через **Capacitor**, а APK
собирается автоматически на GitHub Actions.

### Скачать готовый APK

1. Открой раздел **Releases** репозитория → релиз **«Записной Козёл — Android APK»**.
2. Скачай файл `zapisnoy-kozel-latest.apk` на телефон.
3. На Android разреши установку из неизвестных источников (Настройки →
   Безопасность / «Установка неизвестных приложений»).
4. Открой скачанный файл → **«Установить»**.

> APK пересобирается автоматически при каждом пуше. Вручную запустить сборку:
> вкладка **Actions → Build Android APK → Run workflow**. Готовый файл также
> лежит в артефактах запуска (`zapisnoy-kozel-apk`).

### Собрать APK локально

Нужны JDK 21 и Android SDK (через Android Studio).

```bash
npm run cap:build        # собрать веб-часть и синхронизировать в android/
cd android
./gradlew assembleDebug  # APK → android/app/build/outputs/apk/debug/app-debug.apk
```

Иконки и сплэш генерируются из эмблемы козла:

```bash
npm run icons            # пересоздать иконки/сплэш для всех плотностей
```

## 🌐 Онлайн-игра (аккаунты и столы)

Можно играть с живыми людьми: создать аккаунт, открыть стол (открытый или
по паролю) и сесть за него с другими. Свободные места при старте занимают боты.

Онлайн состоит из двух частей:

- **Сервер** (`server/`) — Node + WebSocket. Аккаунты, лобби, столы и
  **авторитетная** игра (тот же движок, что и в офлайне; чужие руки скрыты).
- **Клиент** — экран «Играть онлайн» подключается к серверу по WebSocket.

### Запустить сервер

```bash
cd server
npm install
npm start            # слушает порт 8080 (переопределяется через PORT)
```

Данные аккаунтов хранятся в `server/data/accounts.json`. Секрет токенов и порт
задаются переменными окружения `AUTH_SECRET` и `PORT`.

### Подключить клиент к серверу

Адрес сервера задаётся на сборке через переменную `VITE_SERVER_URL`:

```bash
VITE_SERVER_URL=ws://localhost:8080 npm run dev      # локально
```

Для APK задайте repo variable `VITE_SERVER_URL` (Settings → Secrets and
variables → Actions → Variables), например `wss://kozel.example.com` — тогда
онлайн заработает и в приложении. Без адреса доступен только офлайн против ботов.

> Сервер нужно где-то хостить (VPS/облако), чтобы он работал постоянно.
> Для боевого режима используйте `wss://` (TLS) и задайте `AUTH_SECRET`.

### Развернуть сервер в Docker

В репозитории есть `server/Dockerfile` (контекст сборки — корень проекта):

```bash
docker build -f server/Dockerfile -t kozel-server .
docker run -p 8080:8080 -e AUTH_SECRET=задайте-секрет kozel-server
```

Образ подходит для любого контейнерного хостинга (Render, Railway, Fly.io,
свой VPS). После деплоя возьмите публичный адрес (`wss://…`) и пропишите его
в `VITE_SERVER_URL`.

## 🗂️ Структура проекта

```
src/
  components/   # UI-компоненты (карта, рука, стол, кнопки, таблица счёта)
  game/         # игровая логика (без UI)
    deck.ts     # колода, масти, символы
    rules.ts    # canPlayCard и проверки ходов
    scoring.ts  # стоимость карт и правило лимита  ← МЕНЯТЬ ОЧКИ ЗДЕСЬ
    engine.ts   # движок: ход, добор, спец-эффекты, конец раунда
    bots.ts     # ИИ ботов (3 уровня)
    sound.ts    # звуковые хуки (WebAudio, без файлов)
    types.ts    # типы Card, Player, GameState, Move, Suit, Rank…
  screens/      # экраны приложения
  store/        # Zustand-хранилища (игра, настройки, статистика)
  styles/       # globals.css (Tailwind)
scripts/sim.ts  # авто-симуляция партий для проверки движка
```

## 🛠️ Где что менять

- **Правила/очки:** `src/game/scoring.ts` (значения карт, правило лимита).
- **Логика спец-карт:** `src/game/engine.ts` → `applySpecialCardEffect`.
- **Поведение ботов:** `src/game/bots.ts`.
- **Дизайн (цвета, тени, шрифты):** `tailwind.config.js` и `src/styles/globals.css`.
- **Дефолтные настройки:** `src/store/settingsStore.ts` (`DEFAULT_SETTINGS`).

## 📜 Правила (кратко)

Цель — первым сбросить все карты. Кладёшь карту по совпадению масти/значения или
даму (меняет масть).

| Карта | Эффект | Переводится? |
|------|--------|--------------|
| 6 | следующий берёт +2 (стопкой) | да, шестёркой |
| 7 | следующий берёт 1 | нет |
| 9 | накрыть той же мастью | да, девяткой |
| Туз | пропуск хода | да, тузом |
| Дама | сменить масть | да, дамой |
| K♠ | следующий берёт 4 | нет |

Очки: 6–10 по номиналу, J=2, Q=3, K=4, A=11. Лимит по умолчанию 101: перебор —
«улетел» (выбываешь), ровно лимит — обнуление до 0.
