# DOFFA — Экран «Зёрна» (тапалка): заметки по реализации

Живой брендовый маскот-экран: игрок тапает по чашке DOFFA и копит **зёрна** —
внутреннюю игровую валюту для входа в матчи.

## Что добавлено

| Файл | Роль |
|---|---|
| `src/features/beans/beansStore.ts` | Экономика зёрен: баланс, энергия (реген), комбо, золотые зёрна, вход в матч. localStorage — **кэш**; истина — сервер (`syncFromServer`). |
| `src/features/beans/DoffaMascot.tsx` | Маскот-чашка (SVG + Framer Motion): idle / tap / tired. Ноги, кофе, пар, рука, брызги — временная качественная версия без внешних ассетов. |
| `src/features/beans/BeansScreen.tsx` | Композиция экрана: HUD (зёрна + энергия), маскот, летящие зёрна и «+N», карточки, кнопки, состояния low-energy / not-enough-beans. |
| `src/components/shared/BottomNav.tsx` | Нижнее меню: Играть · Зёрна · Награды · Профиль. |
| `src/app/App.tsx` | Тип экрана `beans`, роут, показ `BottomNav` на хабовых экранах + отступ под меню. |

## Как встроен

- Новый экран `beans` в роутере `App.tsx` (без react-router — как остальные).
- Нижнее меню показывается на хабах `['home','beans','rewards','profile']`; на игровых/модальных экранах скрыто.
- Вкладка **«Зёрна»** ведёт на `BeansScreen`; «Играть» на экране зёрен → онлайн-лобби (`onPlay → 'online'`).

## Как запустить

```
npm run dev        # затем открыть приложение, нижнее меню → «Зёрна»
npm run lint       # tsc --noEmit
npm run build
```

## Что уже работает

- Тап начисляет зерно, тратит энергию; энергия восстанавливается со временем (тик раз в секунду).
- Комбо при быстрых тапах (множитель ×1..×5, бейдж «Комбо ×N»).
- Редкое **золотое зерно** (~2.5%) — крупный бонус и особый feedback.
- Летящие зёрна к счётчику + всплывающие «+N»; живой счётчик с «поп»-анимацией.
- Idle: «дыхание» чашки, пар, покачивание ног, колыхание кофе.
- Tap: раскачка + сжатие чашки, вскидывание ног, всплеск кофе, вылет руки.
- Состояния: **мало энергии** (уставшая чашка + шторка действий), **недостаточно зёрен** (при «Играть»).
- Гаптика: обычный тап / особый на золотом зерне (через `haptics`).

## Что пока моковое / временное

- **Баланс зёрен и энергия — только клиентский кэш** (localStorage `doffa-crazy8-beans-v1`). Античит и истинный баланс — на сервере (см. ниже).
- Кнопки «Задания», «Бонус дня», «Пригласить друга» — заглушки (навигация/haptics), логики ещё нет.
- Маскот — временная SVG/CSS-версия; финальные арт-ассеты — в `TAP_MASCOT_ASSETS.md`.
- Звук — только «намёк» через haptics; аудио-эффекты тапа/золота подключаются позже.

## Дальнейшие шаги для серверного хранения зёрен

Переиспользовать серверный слой из Этапа 3 (`server/src/services`, `repositories`):
1. Добавить в `DoffaUser` уже есть `cupsBalance` — это и есть зёрна (Cups). Завести
   WS-команды `beans:balance`, `beans:tap` (батч тапов с серверной валидацией
   энергии), `beans:spendEntry` (списание входа в матч).
2. Сервер — источник истины: клиент шлёт «N тапов за интервал», сервер сверяет
   с лимитом энергии и начисляет; ответ → `beansStore.syncFromServer`.
3. Списание входа в матч (`spendEntry`) должно проходить на сервере при `table:start`.
4. Реген энергии считать на сервере по времени (клиентский реген — только для UI).

Пока сервер не подключён — экран полностью функционален локально, а архитектура
уже разделяет «оптимистичный клиентский кэш» и «серверную истину».
