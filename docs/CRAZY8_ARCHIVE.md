# Crazy 8 — архив (скрытый режим)

## Статус

DOFFA Crazy 8 был первым флагманским продуктом этой платформы. С переходом
на **DOFFA Games** и новый основной режим **DOFFA Bean Duel** карточная игра
Crazy 8 **скрыта из публичного интерфейса**, но полностью сохранена в коде —
ничего не удалено, движок, боты, онлайн-лобби, статистика и тесты работают
как прежде за флагом.

## Где находится код

```
src/games/crazy8/
  engine/       # игровая логика (без UI, чистые функции)
    types.ts    # Card, Player, GameState, MoveAction, Suit, Rank…
    deck.ts     # колода, масти, символы
    rules.ts    # canPlayCard, topCard, проверки ходов
    scoring.ts  # стоимость карт, правило лимита очков
    engine.ts   # applyMove, startNextRound, спец-эффекты карт
    bots.ts     # ИИ ботов (3 уровня сложности)
    labels.ts   # короткие подписи действий для UI
  components/   # Card, GameTable, PlayerAvatar, PlayerHand, ScoreBoard, SuitIcon
  screens/      # GameScreen (офлайн), OnlineScreen + OnlineGameScreen (мультиплеер),
                # RulesScreen, StatsScreen, SettingsScreen
  store/        # gameStore.ts (активная партия), statsStore.ts (статистика побед)

server/src/
  lobby.ts      # столы/лобби — импортирует движок из src/games/crazy8/engine
  match.ts      # создание матча, скрытие чужих рук (redactFor)
  protocol.ts   # WS-протокол опирается на GameState/MoveAction из движка Crazy 8
  index.ts      # WS-обработчики table:*/game:* — авторитетная онлайн-игра Crazy 8
```

Общие утилиты, которые раньше лежали рядом с игровой логикой (`src/game/`),
теперь в `src/lib/` (`haptics.ts`, `sound.ts`) — ими пользуются и Crazy 8, и
Bean Duel, и остальной интерфейс.

**Не разделено сознательно:** `src/net/onlineStore.ts` и `src/net/protocol.ts`
совмещают Crazy8-мультиплеер (лобби/стол/ход) с общим слоем кошелька/
зёрен/наград (auth, `beans:*`, `reward:*`) — это один и тот же WS-канал.
Полное разделение на «крейзи8-только» и «общее» — отдельная будущая задача,
если/когда у Bean Duel появится собственный сетевой режим.

## Как включить локально

Клиент:

```bash
VITE_ENABLE_CRAZY8_CLASSIC=true npm run dev
```

На главном экране появятся служебные dev-ссылки «Crazy 8 · офлайн (dev)» и
«Crazy 8 · онлайн (dev)» — без них флаг выключен и режим недостижим из UI.
Дополнительно `App.tsx` жёстко редиректит на главный экран, если состояние
экрана каким-то образом окажется на скрытом экране (`game`, `rules`, `stats`,
`settings`, `online`) при выключенном флаге — защита сверх отсутствия кнопок.

Сервер (для онлайн-режима Crazy 8) запускается как обычно — WS-обработчики
`table:*`/`game:*` не скрыты флагом на сервере (флаг — только клиентский UI
gate); если понадобится жёстко выключить онлайн-Crazy8 и на сервере, это
предстоит сделать отдельно.

## Зависимости

- **React 18 + TypeScript + Framer Motion + Zustand** — как и весь проект,
  ничего специфичного не добавлено.
- **Сервер:** `server/` (Node + `ws`) — авторитетная игра, те же функции
  движка (`applyMove`, `startNextRound`), что и в офлайне.
- Данные аккаунтов/матчей — `server/data/*.json` (файловое хранилище,
  общее с остальной доменной экономикой).

## Как запустить тесты

```bash
npx tsx scripts/sim.ts     # 300 автопартий бот-против-бота — движок Crazy 8
```

Скрипт импортирует напрямую из `src/games/crazy8/engine/` — работает
независимо от того, включён ли `VITE_ENABLE_CRAZY8_CLASSIC` (это чистая
логика без UI).

## Почему режим скрыт

Продукт репозиционирован в **DOFFA Games** с основным режимом **DOFFA Bean
Duel** — короткой skill-based дуэлью, которая лучше подходит формату
«тапни и сыграй за минуту» и не требует стола/ожидания других игроков.
Crazy 8 остаётся полностью рабочим для локальной разработки и как
потенциальный будущий режим платформы, но не рекламируется и не доступен
публично, пока владелец не примет отдельное решение вернуть его в UI.
