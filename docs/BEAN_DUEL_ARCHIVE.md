# Bean Duel — архив (скрытый режим)

## Статус

DOFFA Bean Duel был основным режимом платформы **DOFFA Games**. С переходом
на новый основной режим **DOFFA Defense** (порт комнатного action-roguelite
из одноимённой LibGDX-игры, см. `src/games/doffa-defense/PORT_NOTES.md`)
дуэль **скрыта из публичного интерфейса**, но полностью сохранена в коде —
ничего не удалено, движок, экран и тесты работают как прежде за флагом.

## Где находится код

```
src/games/bean-duel/
  engine.ts           # чистый движок дуэли (без UI); тот же код гоняет
                      # сервер в авторитетном PvP (server/src/duel.ts)
  engine.test.ts      # vitest — детерминизм, рывок, бросок зерна
  BeanDuelScreen.tsx  # React-экран дуэли (указатель + рывок/бросок)
  Duelist.tsx         # маскот-чашка DOFFA
```

Серверный PvP-канал дуэли (`server/src/duel.ts`, WS-сообщения `duel:*` в
`src/net/protocol.ts`) не трогался и флагом не скрывается — флаг, как и у
Crazy 8, только клиентский UI-gate.

## Как включить локально

```bash
VITE_ENABLE_BEAN_DUEL=true npm run dev
```

На главном экране появится служебная dev-ссылка «Bean Duel (dev)».
Дополнительно `App.tsx` жёстко редиректит на главный экран, если состояние
экрана каким-то образом окажется на `bean-duel` при выключенном флаге —
защита сверх отсутствия кнопок (та же схема, что и для Crazy 8).

## Экономика (зёрна / награды)

Bean Duel v1 **не начислял и не списывал** зёрна и DOFFA за матч — это
явно задокументировано в шапке `BeanDuelScreen.tsx` («Экономика: v1 не
списывает и не начисляет зёрна/DOFFA за матч Bean Duel»). Серверные награды
(`reward:*`, см. `src/features/rewards/rewardsStore.ts`) привязаны к
PvP-матчам на сервере и заполняются только ответами сервера.

**Эквивалент в DOFFA Defense:** награды теперь текут через забеги Defense
(серверные сообщения `run:start` / `run:finish`, см.
`server/src/services/runService.ts`). Вход в забег стоит 100 зёрен
(списывает сервер), зачистка комнат даёт зёрна (+5 за комнату, +10 за
мини-босса, +15 за главу — суммы считает только сервер), а DOFFA начисляется
за полное прохождение главы — это эквивалент победы в дуэли: тот же
`computeWinReward` (80/20, burn, общий дневной лимит наград) плюс лимит
наградных глав `MAX_REWARDED_CHAPTERS_PER_USER_PER_DAY` (по умолчанию 5).
Клиент по-прежнему никогда не начисляет зёрна/DOFFA сам — только показывает
подтверждённые сервером суммы; офлайн-забег остаётся играбельным без наград.
Потолки античита описаны в `docs/ANTI_CHEAT.md` (раздел 9).

## Почему режим скрыт

Продукт переходит на **DOFFA Defense** как основной режим — более глубокий
геймплей (комнаты, драфт способностей, боссы) в той же вселенной DOFFA.
Bean Duel остаётся полностью рабочим для локальной разработки, его движок
по-прежнему используется серверным PvP, и режим может вернуться в UI по
отдельному решению владельца.
