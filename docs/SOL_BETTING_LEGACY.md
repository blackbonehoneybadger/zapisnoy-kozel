# Ставки SOL — legacy-механика (выключена по умолчанию)

## Статус

Начиная с окончательной экономической модели DOFFA (зёрна → вход в матч →
DOFFA-награда за подтверждённую сервером победу), ставки в SOL **не входят**
в основной путь игрока. Механика оставлена в коде за отключённым по
умолчанию флагом — на случай будущего возврата после отдельного аудита,
без переписывания заново.

## Как включить (не для продакшена без аудита)

- Сервер: `SOL_BETTING_ENABLED=true` в переменных окружения
  (`server/src/config.ts`). Пока флаг выключен, `table:create` с
  `betLamports` тихо создаёт обычный стол без ставки (сервер игнорирует
  сумму) — платёжный/выплатной код при этом не выполняется.
- Клиент: `VITE_SOL_BETTING_ENABLED=true` при сборке
  (`src/config/features.ts`). Пока флаг выключен, UI ставки скрыт из
  модалки создания стола («Ставка SOL» переключатель и связанные поля).
  Это **только видимость интерфейса** — реальная защита живёт на сервере
  (клиентский флаг нельзя использовать как единственный барьер).

Оба флага независимы и должны включаться вместе для полноценной работы
механики.

## Что делает механика, если включена

1. Хозяин стола указывает размер ставки в SOL при создании стола.
2. Стол требует, чтобы **все места были заняты живыми людьми** — ставка
   не запускается, если за столом остались боты (проверка в
   `server/src/index.ts`, обработчик `table:start`, и в
   `WaitingRoom`/`CreateTableModal` на клиенте). Это защита от ситуации,
   когда игрок рискует SOL против бота, который физически не может
   заплатить свою долю банка.
3. Каждый игрок переводит SOL на кошелёк сервера (`wallet:pay`), сервер
   проверяет транзакцию on-chain (`verifyPayment` в `server/src/solana.ts`)
   и защищается от повторного зачёта одной подписи (`usedSigs`).
4. По окончании партии банк (за вычетом комиссии `PLATFORM_FEE`)
   автоматически отправляется победителю (`handlePayout` в `index.ts`),
   с повторными попытками при сбое RPC.
5. Если игрок выходит после начала ставочной партии, его место занимает
   бот — партия продолжается, ставка не возвращается (см. `leaveTable` в
   `server/src/lobby.ts`).

## Компоненты, которые НЕ удалены

- `server/src/solana.ts` — `sendSol`, `verifyPayment`, `SERVER_WALLET`,
  `PLATFORM_FEE`, `PLATFORM_WALLET`.
- `server/src/lobby.ts` — `betLamports`/`potLamports` на столе,
  `sanitizeBet`, человеко-only проверка в `startGame`-пути.
- `server/src/index.ts` — обработчики `wallet:register`, `wallet:pay`,
  логика `handlePayout`.
- `src/games/crazy8/screens/OnlineScreen.tsx` — блок «Ставка SOL» в `CreateTableModal`
  и связанные предупреждения в `WaitingRoom` (скрыты, не удалены).
- `src/net/protocol.ts` / `server/src/protocol.ts` — поля `betLamports`,
  `potLamports`, сообщения `wallet:*`.

## Перед включением в продакшене потребуется

- Независимый security-аудит платёжного пути (double-spend, race conditions
  при параллельных `wallet:pay`, устойчивость `handlePayout` при падении
  сервера между переводом и рассылкой уведомлений).
- Чёткое юридическое/комплаенс-ревью — ставки реальными деньгами могут
  подпадать под регулирование азартных игр в разных юрисдикциях.
- Решение, остаётся ли эта механика частью основного UI или отдельным
  opt-in режимом с явным предупреждением.
