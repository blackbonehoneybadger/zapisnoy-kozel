# Журнал сжигания DOFFA (burn ledger)

## Статус

**Реальное сжигание НЕ активировано.** `DOFFA_BURN_ENABLED=false` по
умолчанию — как и в продакшене. Ни одна запись журнала сжигания не имеет
статуса `burned`, потому что реальная отправка транзакции сжигания в сеть
пока не реализована (нужны `@solana/spl-token`, доступ к mint-authority или
выделенному burn-кошельку, и подтверждённая инфраструктура выплат — по
аналогии с `SolanaSplTransactionProvider` в `services/transactionProvider.ts`).

## Как это работает сейчас

При каждой подтверждённой сервером наградной победе в DOFFA Bean Duel
(`server/src/services/rewardBudgetService.ts`, вызывается из
`server/src/duel.ts`):

1. Валовая награда делится по `PLAYER_REWARD_PERCENT` / `BURN_PERCENT`
   (по умолчанию 80/20, см. `server/src/config.ts`).
2. Доля игрока идёт в `RewardService.recordMatchResult` — обычная
   доступная награда, которую можно забрать через `reward:claim`.
3. Доля на сжигание записывается в `server/data/burn_ledger.json`
   (`recordBurn(matchId, amount)`) со статусом:
   - **`planned`** — `DOFFA_BURN_ENABLED=false` (сейчас, по умолчанию).
     Сумма учтена и зарезервирована в журнале, но **никакая транзакция не
     создаётся и не имитируется**, поле `txSignature` не выставляется.
   - **`testing`** — `DOFFA_BURN_ENABLED=true`, но реальная отправка в
     сеть ещё не реализована (заготовка на будущее — включение флага само
     по себе пока ничего не отправляет, статус лишь сигнализирует
     «сжигание готовится к тестированию»).
   - **`burned`** — зарезервировано на будущее, когда появится реальная
     реализация отправки burn-транзакции с подтверждённой `txSignature`.
     Сейчас этот статус нигде не выставляется.

Ни при каких условиях журнал не заявляет о сожжённых токенах, если они не
были реально сожжены — это прямое требование задачи: «If burn is not yet
active: do not simulate a transaction; status must be Planned or Testing;
never claim tokens were burned».

## Формат записи

```json
{
  "matchId": "duel-...",
  "amount": 2,
  "status": "planned",
  "createdAt": 1730000000000
}
```

`amount` — доля BURN_PERCENT от валовой награды за этот матч (не доля
игрока — она хранится отдельно в `Reward`/`ClaimRecord`).

## Что нужно для реального включения (будущая задача)

1. Выделенный кошелёк для сжигания (`DOFFA_BURN_WALLET_PRIVATE_KEY`,
   **только на сервере**, никогда в клиенте/APK/VITE_*/GitHub).
2. Реализация отправки burn-инструкции SPL-токена (`@solana/spl-token`,
   `createBurnInstruction` или перевод на insert-адрес без ключа с
   последующим сжиганием — по решению владельца).
3. Обновление `recordBurn`/статуса записи на `burned` только ПОСЛЕ
   подтверждения транзакции в сети — та же идемпотентность и защита от
   двойной обработки, что уже реализована в `ClaimService` для наград
   игроку (см. `docs/ANTI_CHEAT.md`).
4. Мониторинг остатка сжигающего кошелька и статуса сети — аналогично
   `DOFFA_MIN_HOT_WALLET_BALANCE` для наградного кошелька.
