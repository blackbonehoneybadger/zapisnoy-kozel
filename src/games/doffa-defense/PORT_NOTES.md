# DOFFA Defense — заметки о порте

Порт комнатного режима (vertical slice) из Java/LibGDX-версии
(`doffa-defense/core/src/main/java/com/doffa/defense/`) в чистый TypeScript
(`engine.ts`, без DOM) + React/canvas экран (`DoffaDefenseScreen.tsx`).

## Что перенесено 1:1

- Движение + «стоишь = авто-атака ближайшего врага» (PlayerController:
  220 px/s, грейс 0.15с, интервал 0.35с).
- 4 врага с точными статами (EnemyCatalog): химический курьер (CHASE),
  токсичный метатель (KEEP_DISTANCE), бронированный варщик (TANK/CHASE),
  мутировавший делец (элита, KEEP_DISTANCE); ELITE-ростер = базовый +
  дубликат элиты.
- KEEP_DISTANCE сближается из-за пределов реальной дальности атаки
  (регрессия R-Stage5 сохранена в тестах).
- Сидированная генерация главы (RoomGenerator) **включая незакоммиченные
  баланс-гарды**: index-1 никогда не ELITE; RECOVERY гарантированно строго
  до мини-босса (с ремонтом инварианта «не подряд»); две небоевые комнаты
  не идут подряд; веса 0.55/0.20/0.12/остаток; 10 шаблонов комнат биома.
- Мини-босс «Токсичный надзиратель» (1 фаза) и босс главы «Химический
  Варщик» (2 фазы, 3 атаки) с телеграфами: намотка → разрешение по позиции
  игрока (уклонение из радиуса — полноправный исход), пауза-кадр на смену
  фазы, отмена телеграфа при смене фазы.
- Драфт 1-из-3 после боевых комнат и в сокровищницах; все 26 карт каталога
  с подключёнными эффектами (урон/скорость/крит/снаряды/пробитие/поджог/
  защита/реген/похищение/скорострельность/неуязвимость рывка). Поджог —
  refresh-not-stack. Пробитие тратит заряды. Крит/похищение через общий
  путь нанесения урона.
- HP/смерть игрока, реген, множители лоадаута с теми же капами
  (крит ×1.5+, мин. получаемый урон 0.2, мин. интервал огня 0.4,
  макс. похищение 0.9).
- ESC-пауза (из незакоммиченной версии RoomScreen): замораживает всю
  симуляцию; ESC/Enter — продолжить, Q — выйти в меню.
- Русские строки UI — те же, что в Java-версии («ПАУЗА», «БЕЗОПАСНАЯ
  КОМНАТА», «ЗАВЕДЕНИЕ ЗАКРЫТО», «ГЛАВА ПРОЙДЕНА», заголовки драфта и
  подписи типов комнат).

## Отличия, о которых нужно знать

- **PRNG:** Java использует `java.util.Random`; порт использует mulberry32.
  Поток случайных чисел бит-в-бит с Java **не** совпадает (другой алгоритм),
  но детерминизм «один сид → одна глава/спавн/драфты/криты» сохранён
  внутри TS-движка и покрыт тестами. Сиды забега — те же числа
  (20260717_1/2/3).
- **Фиксированный шаг:** симуляция шагает строго по 1/60с через
  накопительный `advance(dt)` — в Java тик привязан к кадру LibGDX.

## Сознательные сокращения (web-непрактично / вне скоупа)

- **Реклама и возрождение за рекламу** (AdPolicy, rewarded revive на R):
  на веб-платформе рекламного слоя нет. Смерть → только «Ещё раз / В меню».
- **Мета-прогрессия и таланты** (MetaProgressionService, TalentCatalog,
  HeroCatalog, другие герои): в Java это отдельные экраны и постоянное
  хранилище; вертикальный срез играется бариста с базовым лоадаутом.
- **Аудио и вибрация** (AudioCueService, platformServices.vibrate): заменены
  существующим `haptics` проекта там, где это уместно.
- **Награды/зёрна:** не начисляются — поведение 1:1 повторяет Bean Duel v1
  (см. `docs/BEAN_DUEL_ARCHIVE.md`, раздел «Экономика»).
