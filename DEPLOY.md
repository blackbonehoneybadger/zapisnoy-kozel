# Деплой DOFFA Games

Приложение состоит из двух частей:

1. **Фронтенд (PWA)** — React + Vite, статика. Хостится на **Vercel**.
2. **Онлайн-сервер** — Node.js WebSocket (вход через кошелёк, лобби, столы,
   авторитетная игра, ставки и выплаты Solana). Хостится на **Railway**.

---

## 1. Онлайн-сервер (Railway)

### 1.1. Создать сервис
1. [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub repo**
2. Выбрать репозиторий проекта DOFFA Games (текущее имя репозитория —
   `blackbonehoneybadger/zapisnoy-kozel`; готовится переименование в
   `doffa-games`, см. `docs/REPOSITORY_RENAME.md`)
3. Railway найдёт `railway.toml` и соберёт образ из `server/Dockerfile`
   (контекст сборки — корень репозитория, серверу нужен движок Crazy 8 из
   `src/games/crazy8/engine`).

### 1.2. Переменные окружения (Variables)

| Переменная | Значение | Обязательна |
| --- | --- | --- |
| `SOLANA_NETWORK` | `mainnet-beta` (или `devnet` для тестов) | да |
| `SOLANA_PRIVATE_KEY` | приватный ключ горячего кошелька, JSON-массив `[..]` | да на mainnet |
| `PLATFORM_FEE` | `0.05` (5% комиссии с банка) | нет (дефолт 0.05) |
| `PLATFORM_WALLET` | публичный адрес кошелька для комиссии/дохода | нет |
| `SOLANA_RPC_URL` | свой RPC-эндпоинт (если есть) | нет |
| `PORT` | Railway задаёт сам | нет |

> ⚠️ `SOLANA_PRIVATE_KEY` — **секрет**. Хранится только в Railway Variables,
> никогда не коммитится в репозиторий (см. `.gitignore`).

На `mainnet-beta` сервер **не запустится** без `SOLANA_PRIVATE_KEY` — это
защита от игры на реальные деньги с пустым временным кошельком.

### 1.3. Пополнить горячий кошелёк
Горячий кошелёк (адрес которого соответствует `SOLANA_PRIVATE_KEY`) собирает
ставки и платит выигрыши. Ему нужен небольшой запас SOL на газ транзакций
(~0.05–0.1 SOL). Без газа выплаты будут откладываться (есть авто-повтор).

### 1.4. Публичный домен
Railway → **Settings** → **Networking** → **Generate Domain**.
Полученный адрес `*.up.railway.app` — это и есть `wss://*.up.railway.app`.

### 1.5. Проверка
- Логи при старте: `◎ Solana: сеть mainnet-beta, кошелёк сервера …, комиссия 5%`
- Health-check: `GET https://<домен>/health` → `{"ok":true,...}`

## 2. Фронтенд (Vercel)

### 2.1. Переменные окружения (Project Settings → Environment Variables)

| Переменная | Значение | Назначение |
| --- | --- | --- |
| `VITE_SERVER_URL` | `wss://<домен Railway>` | адрес онлайн-сервера |
| `VITE_SOLANA_NETWORK` | `mainnet-beta` | сеть для клиентских платежей |

После задания переменных — **Redeploy**, чтобы сборка их подхватила.

> ⚠️ Ставки SOL, описанные ниже, — legacy-механика, выключенная по
> умолчанию (`SOL_BETTING_ENABLED=false` на сервере,
> `VITE_SOL_BETTING_ENABLED=false` на клиенте). Основной путь платформы —
> зёрна → вход в матч → DOFFA-награда за победу, без ставок реальными
> деньгами. См. `docs/SOL_BETTING_LEGACY.md`.

### 2.2. Архитектура платежей (только если SOL_BETTING_ENABLED=true)

```
Игроки ──ставки──▶ [Горячий кошелёк сервера]   (SOLANA_PRIVATE_KEY на Railway)
                          ├──95%──▶ Победитель
                          └──5%───▶ [PLATFORM_WALLET]   (доход площадки)
```

- **Политика ставок: возвратов нет.** После оплаты ставка не возвращается ни
  при каких условиях (выход до старта или в середине партии = форфейт).
- Банк фиксируется при старте партии (`potLamports`) и не уменьшается, если
  кто-то вышел в середине — его ставка достаётся победителю (форфейт).
- Выплата защищена от двойного срабатывания (`paidOut`) и повторно проверяет
  подписи (анти-реплей). При сбое RPC выплата авто-повторяется.

## 3. Безопасность

- Вход только подписью кошелька (ed25519, nonce challenge-response) — паролей нет.
- `verifyPayment` проверяет, что SOL **реально пришли на кошелёк сервера** в нужном объёме.
- Анти-реплей подписей платежей (persisted в `server/data/used_sigs.json`).
- Rate limiting WebSocket: 60 сообщений/10с на сокет + лимит 16 КБ на сообщение.
- Секреты и состояние сервера (`server/data/`, `.env`) — в `.gitignore`.

---

## 4. Авто-обновление PWA

Клиент использует Service Worker (`registerType: 'autoUpdate'`) + периодическую
проверку обновлений (раз в 30 мин и при возврате во вкладку). Установленное
приложение обновляется само, без переустановки.
